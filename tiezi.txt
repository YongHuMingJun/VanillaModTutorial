[size=24px][b]§1 简介[/b][/size]
[size=16px]原版模组(vanilla mod)一般指在不修改Minecraft游戏本体的前提下，通过命令方块、一键命令(OOC)、数据包(datapack)、资源包(resourcepack)等方式对游戏的可玩性做出修改。而自 Minecraft Java 版 1.13 起的数据包概念问世之后，原版模组的制作已变得十分便捷。然而纵观论坛，原版模组仍然不够繁荣。因此我将为首次接触此概念的玩家做一个简单的入门，以期抛砖引玉。[/size]

[size=16px]部分段落常为部分读者所熟知，若如此可直接跳至自己所需段落。对于首次接触者，请耐心阅读。[/size]

[size=16px]本文适用 Minecraft Java 版 1.13 及更高版本。系统环境为 Windows10，其它环境下有较小的差异。[/size]

[size=20px][b]§1.2 工具准备[/b][/size]

[size=16px][b]文本编辑器[/b][/size]

[size=16px]我们所涉及的所有文本文件，包括函数文件(.mcfunction)、JSON文件(.json, .mcmeta)，均需使用 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]UTF-8[/backcolor][/font] 编码格式。以下列出的文本编辑器默认均为 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]UTF-8[/backcolor][/font] 编码，可以在编辑页面的右下角状态栏看到。[/size]

[size=16px][list]
[*][b][url=https://code.visualstudio.com/]VS Code[/url][/b] 下载安装后，可以安装如下插件：[size=16px][list]
[*][i]Chinese (Simplified) Language Pack for Visual Studio Code[/i] 支持中文语言
[*][i]datapack helper plus (JSON)[/i] 支持 JSON 语法补全，高亮等
[*][i]language-mcfunction[/i] 支持 mcfunction 语法和高亮
[*][i]mcfunction[/i] 支持 mcfunction 语法和高亮[/list][/size]
[*][b]记事本[/b] 为 Windows 系统自带。Windows10 最新版本的记事本已经默认是 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]UTF-8[/backcolor][/font] 编码了。点击[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]查看->状态栏[/backcolor][/font]可以在右下角状态栏看到，因此可以正常使用。旧版本的记事本仍然不是，请勿使用。
[*][b][url=https://notepad-plus-plus.org/]Notepad++[/url][/b]
可从 [url=http://www.mcbbs.net/thread-806816-1-1.html]mcfunction 的语言样式和自动补全[/url]帖中下载相关文件并导入，另外请将 Notepad++ 中[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]设置->首选项->其它->自动检测字符编码[/backcolor][/font]选项关闭。[/list][/size]
[size=16px]右下角也可以看到 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]LF[/backcolor][/font] 或 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]CRLF[/backcolor][/font]，分别表示两种换行符，二者均可正常使用，建议使用 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]LF[/backcolor][/font]。[/size]

[size=16px]注意不要误选了 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]UTF-8 with BOM[/backcolor][/font] 格式。[/size]

[size=16px][b]压缩工具[/b][/size]

[size=16px]数据包和资源包均可以为文件夹或 zip 压缩文件格式。游戏本体和模组本体的 jar 文件也需要压缩工具来打开。[/size]

[size=16px][list]
[*][b]Windows 资源管理器[/b] Windows10 系统自带 zip 格式的压缩和解压缩。zip 格式可以直接打开，压缩则用[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]选择->右键->发送到->压缩文件夹[/backcolor][/font]。

[*][b]winrar[/b] 付费软件。[/list][/size]

[size=20px][b]§1.3 文件类型[/b][/size]

[size=16px]我们先熟悉下所涉及的文件类型，具体用途见后文。为便于查看文件后缀，请将[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]文件->文件夹选项->查看->隐藏已知文件类型的扩展名[/backcolor][/font]选项去掉勾选。前三者为纯文本文件，可通过新建文本文件并修改后缀来创建。[/size]

[size=16px][list]
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0].json[/backcolor][/font] 进度、战利品表、标签、模型、语言文件、断言、配方均为该格式。
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0].mcmeta[/backcolor][/font] 格式与 json 相同，仅用于记录资源包和数据包的信息(版本和描述)。
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0].mcfunction[/backcolor][/font] 即函数文件，每一行都是一条单独的可执行的命令。
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0].nbt .mca[/backcolor][/font] 使用 nbt 查看器打开，打开后为树状结构。世界生成、玩家、结构、地图等信息均为 nbt 格式，区块为 mca 格式。
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0].png[/backcolor][/font] 材质文件格式必须为png。
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0].zip[/backcolor][/font] 双击打开或右键选择使用压缩工具打开，通常是资源包和数据包。
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0].jar[/backcolor][/font] 右键选择使用压缩工具打开，通常是 Minecraft 或 mod 本体。下面是快照 18w31a 的游戏本体 jar 文件的文件内容。
[img]https://i.loli.net/2018/08/03/5b63c9784b438.png[/img][/list][/size]

[size=20px][b]§1.4 文件结构[/b][/size]

[size=16px]打开[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0].minecraft[/backcolor][/font]所在的文件夹，这通常位于你的启动器目录下。如果启动器中设置为各版本独立，则类似的文件结构位于[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]versions/版本号[/backcolor][/font]下。[/size]

[size=16px][list]
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]assets[/backcolor][/font] Minecraft 的资源文件。可在此找到所有游戏内翻译文本的键和翻译，之后便可通过加载资源包来修改。打开 indexes 中对应的 JSON 文件，查找 zh_cn 对应的 hash 值，然后在 objects 中找到相应文件，用文本编辑软件打开即可看到相应的文本。
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]resourcepacks[/backcolor][/font] 资源包文件夹，每个子文件夹或 zip 对应一个资源包。
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]screenshots[/backcolor][/font] 游戏截图。
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]logs[/backcolor][/font] 日志信息。可打开 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]latest.log[/backcolor][/font] 来查看加载资源包和数据包时的错误信息，包括错误的文件名称、位置、错误的行列数等，这对于我们开发原版模组是十分重要的。
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]options.txt[/backcolor][/font] 游戏设置，诸如音量、视距等。[/list][/size]
[size=16px][img]https://i.loli.net/2018/08/03/5b63c978482ce.png[/img][/size]

[size=16px][list]
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]saves[/backcolor][/font] 地图存档，我们从网上下载的地图一般就是解压到该文件夹。[size=16px][list]
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]advancements[/backcolor][/font] 玩家进度和完成时间。
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]playerdata[/backcolor][/font] 玩家属性，可修改玩家的位置、复活点、飞行速度等。
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]stats[/backcolor][/font] 玩家统计信息。
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]data[/backcolor][/font] 地图、记分板、村庄等内容。[size=16px][list]
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]idcounts.dat[/backcolor][/font] 记录了当前已使用的地图数量，修改后新打开的空地图会以此开始计数，这可以避免模组自定义的地图被玩家无意修改。
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]map_数字.dat[/backcolor][/font] 相应的地图信息。
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]scoreboard.dat[/backcolor][/font] 记分板、记分板的值、组。
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]command_storage_命名空间[/backcolor][/font] 存取玩家自定义存储区 (storage) 的nbt。
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]village, stronghold, mineshaft, raids[/backcolor][/font] 村庄、要塞、废弃矿井、袭击的信息。[/list][/size]
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]datapacks[/backcolor][/font] 数据包文件夹，每个子文件夹或 zip 对应一个数据包。
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]DIM1、DIM-1、region[/backcolor][/font] 末地、下界和主世界的区块信息。.mca文件可用 nbtexplorer 打开并修改相应的区块内容，但是极为不便。建议使用 mcedit 等工具来编辑。
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]level.dat[/backcolor][/font] 世界信息，包含了种子、生成类型、难度、出生点、世界边界、游戏规则等内容。
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]generated[/backcolor][/font] 手动保存的结构，保存后将其移动至数据包内方可使用。[/list][/size][/list][/size]

[size=24px][b]§2 数据包[/b][/size]

[size=16px]数据包文件层次为[/size]

[code]datapacks
    数据包名称或数据包名称.zip
        pack.mcmeta
        data
            命名空间
                advancements
                functions
                loot_tables
                recipes
                structures
                tags
                    blocks
                    items
                    fluids
                    functions
                    entity_types[/code]
[size=16px][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]data[/backcolor][/font] 下所有文件和文件夹需使用小写英文、数字或-,_来命名，[b]不可使用大写字母[/b]，所有文本文件使用 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]UTF-8[/backcolor][/font] 编码。数据包可以为文件夹格式或 zip 格式，发布时可将所有内容压缩为一个 zip 文件。[/size]

[size=16px]压缩和解压的时候，注意文件层次，应当打开 zip 文件就可以看到数据包的 mcmeta 文件和 data 文件夹。稳妥的做法是[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]打开文件夹->全选->右键->发送到->压缩文件夹[/backcolor][/font]。[/size]

[size=16px]游戏内输入 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]\datapack list[/backcolor][/font] 可以列出所有的数据包。[/size]

[size=16px][b]原版数据包 (Vanilla)[/b] Minecraft 本体文件([font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]版本号/版本号.jar[/backcolor][/font])中的 data 文件结构与数据包下 data 文件结构相同，其中包含了原版进度、配方、结构、战利品表等内容。这也是制作数据包的文件样板。[/size]

[size=16px]进度、战利品表、标签、模型、语言文件、断言、配方均为 JSON 格式文件。JSON 文件包括一对根括号{}，包含类似 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]"abc":"def"[/backcolor][/font] ([font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]键:值[/backcolor][/font])的对。相同的键后者会覆盖前者。键为需引号圈住的字符串，值可以为[/size]

[size=16px][list]
[*][b]布尔型[/b] 例如 [size=16px][list]
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]"is_on_fire": false[/backcolor][/font] 
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]"is_baby": true[/backcolor][/font][/list][/size]
[*][b]数值[/b] 例如 [size=16px][list]
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]"count": 2[/backcolor][/font]
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]"chance": 0.025[/backcolor][/font][/list][/size]
[*][b]字符串[/b] 使用双引号圈住，可以使用颜色代码如§6、换行符\n。例如 [size=16px][list]
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]"item": "minecraft:shears"[/backcolor][/font]
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]"condition": "minecraft:match_tool"[/backcolor][/font]
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]"layer0": "cpp:crop/bauhinia_seeds"[/backcolor][/font]
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]"description": "§6更多的合成 1.10\n§6by ruhuasiyu"[/backcolor][/font][/list][/size]
[*][b]列表[/b] 使用中括号圈住，例如 [size=16px][list]
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]"chances": [0.05, 0.0625, 0.083333336, 0.1][/backcolor][/font]
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]"items": [{"item": "minecraft:zombie_head"}, {"item": "minecraft:skeleton_skull"}, {"item": minecraft:wither_skeleton_skull"}, {"item": "minecraft:creeper_head"}][/backcolor][/font]
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]"requirements":[["wing_of_sky", "heart_of_crystal", "nova_of_fire"]][/backcolor][/font][/list][/size]
[*][b]JSON 对象[/b] 使用大括号圈住，例如[size=16px][list]
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]"count": { "min": 1, "max": 2 }[/backcolor][/font]
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]"modifiers": [{"name": "legs_armor", "attribute": "generic.armor", "operation": "addition", "amount": 4, "slot": "legs" }][/backcolor][/font][/list][/size][/list][/size]
[size=16px]编写时，使用空格或制表符(Tab)缩进，以便于查看括号匹配和层次。[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0].mcmeta[/backcolor][/font] 文件也是 JSON 文件，所以格式是相同的。[/size]

[size=16px]wiki 上有关页面使用了 NBT 的数据类型标注，但其实并不适用于 JSON 格式。[/size]

[size=20px][b]§2.1 pack.mcmeta[/b][/size]

[size=16px]Minecraft 通过该文件来识别数据包，因此该文件是不可或缺的。例：[/size]

[code]{
    "pack": {
        "pack_format": 4,
        "description": "§6数据包名称\n§a简要介绍 §6by 某某作者"
    }
}[/code]
[size=16px][list]
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]pack_format[/backcolor][/font] 为数据包版本，必须为4。
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]description[/backcolor][/font] 为数据包描述，可以是单个的字符串或一个 raw json 对象。在输入 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]/datapack list[/backcolor][/font] 命令后，鼠标移至数据包名称上会显示 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]description[/backcolor][/font] 信息。[/list][/size]

[size=20px][b]§2.2 命名空间[/b][/size]

[size=16px]命名空间(namespace)为玩家自定义的、可操作的空间。使用独立的命名空间也有利于解决和他人的冲突。[/size]

[size=16px]数据包下可以有多个命名空间，如果不同数据包中有相同的命名空间，则其中相同的文件名内容会根据加载先后顺序被覆盖。特别地，原版内容被保存在 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]minecraft[/backcolor][/font] 命名空间，想要修改和替换原版的内容只需在你的数据包内建立 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]minecraft[/backcolor][/font] 命名空间和相应的同名文件并修改即可。[/size]

[size=16px]标签文件，即 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]tags[/backcolor][/font] 中的文件内容默认追加而不是覆盖。因此标签文件是用来解决数据包冲突、联动的有力工具。[/size]

[size=16px]数据包加载顺序并不固定，因此不要在不同数据包中同一命名空间下使用相同的文件名。原版 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]minecraft[/backcolor][/font] 下内容总是最先被加载。可以在游戏内通过手动开启/关闭来调整加载次序。[/size]

[size=16px]进度、战利品表、函数、结构、标签、断言等文件的调用格式均为[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]命名空间:文件夹/文件名[/backcolor][/font]对应[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]命名空间/xxx/文件夹/文件名.后缀[/backcolor][/font]文件，[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]xxx[/backcolor][/font] 为 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]advancements, loot_tables, functions, strutures, tags/blocks, tags/items[/backcolor][/font] 等。[/size]

[size=16px]若命名空间为 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]minecraft[/backcolor][/font]，则可直接省略 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]minecraft:[/backcolor][/font]。[/size]

[size=20px][b]§2.3 进度[/b][/size]

[size=16px]可参考[/size]

[size=16px][list]
[*][url=http://www.mcbbs.net/thread-685310-1-1.html]【烯方的那一套理论】猴子都能学会的自定义advancement！[/url]
[*][url=https://minecraft-zh.gamepedia.com/%E8%BF%9B%E5%BA%A6]进度 - Minecraft Wiki，最详细的官方我的世界百科[/url][/list][/size]
[size=16px]进度(advancements)为游戏内检测玩家行为，触发后完成进度并执行奖励的系统。游戏内按 L 即可查看。[/size]

[size=16px]该文件夹或子文件夹下每一个 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]root.json[/backcolor][/font] 文件都对应一个进度页面选项卡，因此可通过文件夹来分离不同的选项卡。[/size]

[size=16px]例：[/size]

[code]{
    "display": {
        "icon": {
            "item": "minecraft:carrot_on_a_stick",
            "nbt": "{CustomModelData:10}"
        },
        "title":{
            "translate": "advancements.cpp.root.title"
        },
        "description": {
            "translate": "advancements.cpp.root.description"
        },
        "frame": "goal",
        "background": "cm:bg.png"
    },
    "criteria": {
        "dream_wand": {
            "trigger": "minecraft:inventory_changed",
            "conditions":{
                "items":[
                    {
                        "nbt":"{id:\"cpp:dream_wand\"}"
                    }
                ]
            }
        }
    },
    "rewards": {
        "function": "cpp:adv/root",
        "recipes": ["minecraft:crafting_table","cpp:strange_block"],
        "experience": 200
    }
}[/code]
[size=16px]该例子进度[/size]

[size=16px][list]
[*]图标为胡萝卜钓竿的自定义模型数据为10的模型
[*]标题为翻译文本 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]advancements.cpp.root.title[/backcolor][/font]
[*]描述为翻译文本 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]advancements.cpp.root.description[/backcolor][/font]
[*]样式为圆形边框
[*]背景为资源包中 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]cm/textures/bg.png[/backcolor][/font] 文件
[*]检测条件为玩家背包含有 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]tag:{id:"cpp:dream_wand"}[/backcolor][/font] 的物品
[*]奖励包括函数 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]cpp:adv/root[/backcolor][/font]，200点经验，解锁配方 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]"minecraft:crafting_table"[/backcolor][/font] 和 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]"cpp:strange_block"[/backcolor][/font]。[/list][/size]
[size=16px]具体含义为[/size]

[size=16px][list]
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]display[/backcolor][/font] 进度的样式。若无，则该进度不会出现在进度页面。[size=16px][list]
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]icon[/backcolor][/font] 进度的图标。
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]item[/backcolor][/font] 图标物品名称。
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]nbt[/backcolor][/font] 图标物品 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]nbt[/backcolor][/font]，物品的材质模型可能会根据 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]nbt[/backcolor][/font] 不同而不同。[/list][/size]
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]title[/backcolor][/font] 进度的名称，可以是单个的字符串或一个 raw json 对象，样例中 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]translate[/backcolor][/font] 文本是会随着语言改变的文本，参考[url=thread-691700-1-1.html?cp=3]§3 资源包[/url]。
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]description[/backcolor][/font] 进度的描述，和进度的名称格式相同。
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]frame[/backcolor][/font] 进度的类型，分为 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]task[/backcolor][/font](任务 方形边框)，[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]challenge[/backcolor][/font](挑战 尖形边框)，[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]goal[/backcolor][/font](目标 圆形边框)。
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]background[/backcolor][/font] 仅根进度有，决定该选项卡的背景图，值为资源包内图片，可以为 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]minecraft[/backcolor][/font] 自带或其它资源包。
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]parent[/backcolor][/font] 除根进度外的进度，记录其父进度。
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]criteria[/backcolor][/font] 判据，其下为判据名称、触发器和触发器的条件。判据可以有多个。
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]requirements[/backcolor][/font] 判据的条件格式，缺省时默认所有条件都需要满足。格式为一些或的与(合取范式)，例如 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]"requirements": [["a","b"],["c","d"]][/backcolor][/font] 表示当 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]a[/backcolor][/font] 和 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]b[/backcolor][/font] 至少有一个满足且 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]c[/backcolor][/font] 和 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]d[/backcolor][/font] 至少有一个满足时，会成功触发。
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]rewards[/backcolor][/font] 奖励[size=16px][list]
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]function[/backcolor][/font] 奖励函数，完成进度时执行之。不可为函数标签。
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]loot[/backcolor][/font] 奖励战利品表。
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]recipes[/backcolor][/font] 解锁配方。配方也需要有相应进度来解锁，通常检测玩家拥有配方材料后触发，参考[url=thread-691700-1-1.html?cp=2]§2.7 配方[/url]。
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]experience[/backcolor][/font] 奖励经验，建议仅在高难度进度完成时给予。[/list][/size][/list][/size]
[size=16px]进度配合奖励函数剥夺玩家该进度，则该进度可反复触发。可用于诸如检测玩家生物群系、饮食等需要循环检测的情形。[/size]

[size=16px]游戏内或函数内使用 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]advancement[/backcolor][/font] 命令来手动给予或剥夺进度。[/size]

[size=20px][b]§2.4 函数[/b][/size]

[size=16px]函数(functions)的文件后缀为 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]mcfunction[/backcolor][/font]，每一行表示一个单独的命令，无需/开头。执行时会按顺序依次执行，使用[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]#[/backcolor][/font]开头的行为注释。[/size]

[size=16px]函数可[/size]

[size=16px][list]
[*]在游戏内或函数文件中使用诸如 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]function namespace:foo/bar[/backcolor][/font] 命令来依次执行函数的每一条命令。
[*]在进度完成后奖励函数。[/list][/size]

[size=20px][b]§2.7 配方[/b][/size]

[size=16px]配方(recipes)包括工作台合成、切石机合成、熔炉烧炼、高炉烧炼、烟熏炉烧炼、营火烧炼，**不支持带 nbt 标签。具体格式见[url=thread-691700-1-1.html?cp=32]§6 合成与烧炼配方[/url]。想要实现带 nbt 的合成需要藉由其它方式，见[url=thread-691700-1-1.html?cp=6]§6 合成与烧炼配方[/url]。[/size]

[size=16px]在默认情形(限制配方关闭)下，玩家总是可以使用所有的配方，但是未获得的不会在配方书中显示，因此建议为你的自定义配方添加合适的进度来获取该配方。[/size]

[*][url=https://minecraft-zh.gamepedia.com/结构方块]结构方块 - Minecraft Wiki，最详细的官方我的世界百科[/url][/list][/size]
[size=16px]结构(structures)文件为 nbt 文件，后缀为 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]nbt[/backcolor][/font]。结构存储了一个长方体区域的方块和实体信息，玩家可以创建、保存、调用这样的结构。[/size]

[size=16px]在游戏中获得结构方块(structure_block)后，放置好所需要的方块结构和实体，然后放置结构方块，设置为保存模式，调整好大小，输入名称，点保存即可（注意区分是否需要实体）。然后从 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]generated[/backcolor][/font] 文件夹复制到数据包内，在结构方块中输入"命名空间:文件夹/文件"格式加载（注意区分是否需要实体）。[/size]

[size=16px]我们也可以在函数中加载，先放置结构方块，然后放置并清除红石块即可。这个技巧配合战利品表等，可用于随机在世界生成结构。[/size]

[size=16px]例：[/size]

[code]setblock ~ ~ ~ minecraft:structure_block{posX:-1,posY:0,posZ:-1,name:"cpp:build/enchanting_room",mode:"LOAD"}
setblock ~ ~1 ~ minecraft:redstone_block
setblock ~ ~1 ~ minecraft:air[/code]

[size=20px][b]§2.9 标签[/b][/size]

[size=16px]标签(tags)根据子文件夹不同分为5种。Minecraft 中有很多内容都叫做标签，注意区分它们。[/size]

[size=16px]例如方块标签 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]cpp/tags/blocks/fluid.json[/backcolor][/font][/size]

[code]{
    "replace": false,
    "values": [
        "minecraft:air",
        "minecraft:cave_air",
        "minecraft:void_air",
        "minecraft:bubble_column",
        "minecraft:water",
        "minecraft:lava"
    ]
}[/code]
[size=16px]表示空气、气泡柱、水、熔岩。[/size]

[size=16px]标签中 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]replace[/backcolor][/font] 可以为[/size]

[size=16px][list]
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]false[/backcolor][/font] 表示追加当前 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]values[/backcolor][/font] 至该标签
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]true[/backcolor][/font] 表示覆盖之前原版和其它数据包添加至该标签的内容。[/list][/size]
[size=16px]默认为 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]false[/backcolor][/font]。其它标签也是如此。[/size]

[size=16px]标签中亦可引用其它标签，例如 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]minecraft/tags/blocks/logs.json[/backcolor][/font][/size]

[code]{
    "values": [
        "#minecraft:dark_oak_logs",
        "#minecraft:oak_logs",
        "#minecraft:acacia_logs",
        "#minecraft:birch_logs",
        "#minecraft:jungle_logs",
        "#minecraft:spruce_logs"
    ]
}[/code]
[size=16px]就包含了所有6种原木、木头、去皮原木、去皮木头。[/size]

[size=16px][b]§2.9.1 方块标签[/b][/size]

[size=16px]位于 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]tags/blocks[/backcolor][/font] 文件夹下，用于在命令中用于检测/调用多个方块。例：当玩家眼睛位置不是流体(空气、水等)时，输出 I'm stucked![/size]

[size=16px]并执行命令[/size]

[code]execute as @a at @s anchored eyes unless block ^ ^ ^ #cpp:fluid run say I'm stucked![/code]
[size=16px]因为命令执行地点为实体的脚，所以我们使用 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]anchored eyes[/backcolor][/font] 来使得局部坐标的位置变为玩家的眼睛。[/size]

[size=16px]调试模式(F3)下，玩家指向方块时会显示含有该方块的所有方块标签；[/size]

[size=16px][b]§2.9.2 物品标签[/b][/size]

[size=16px]位于 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]tags/items[/backcolor][/font] 文件夹下，用于[/size]

[size=16px][list]
[*]在 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]clear[/backcolor][/font] 命令中用于清除多个不同 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]id[/backcolor][/font] 物品
[*]在配方文件中表示合成/烧炼时所需的物品可以为标签中的任一种
[*]进度判定
[*]战利品表类型[/list][/size]
[size=16px][b]§2.9.3 函数标签[/b][/size]

[size=16px]位于 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]tags/functions[/backcolor][/font] 文件夹下，用于在 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]function[/backcolor][/font] 命令中一次性按次序执行多个函数。[/size]

[size=16px][b]§2.9.4 实体类型标签[/b][/size]

[size=16px]位于 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]tags/blocks[/backcolor][/font] 文件夹下，用于在选择器 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]@e[/backcolor][/font] 的选项 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]type[/backcolor][/font] 中指定多种不同实体，例如 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]@e[type=#cpp:hostiles][/backcolor][/font]。[/size]

[size=16px][b]§2.9.5 流体标签[/b][/size]

[size=16px]位于 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]tags/blocks[/backcolor][/font] 文件夹下。[/size]

[size=16px][b]§2.9.6 预设标签[/b][/size]

[size=16px][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]minecraft[/backcolor][/font] 命名空间中预设了一些标签。其中[/size]

[size=16px][list]
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]minecraft/tags/functions/load.json[/backcolor][/font] 中的函数会在加载时被执行一次，我们常称之为加载函数、load函数。用于初始化的函数应当添加至该标签。
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]minecraft/tags/functions/tick.json[/backcolor][/font] 中的函数每刻会被执行一次，我们常称之为循环函数、主函数、tick函数。需要高频执行的函数应当添加至该标签。[/list][/size]
[size=16px]利用预设标签还可以实现更多燃料、改变植物种植所需的方块，甚至改变水和熔岩的性质，可参见[url=http://www.mcbbs.net/thread-811990-1-1.html]minecraft预设tag以及实现随机方块功能[/url]以及[url=https://minecraft-zh.gamepedia.com/%E6%A0%87%E7%AD%BE]标签 - Minecraft Wiki，最详细的官方我的世界百科[/url]。[/size]

[size=16px]例如：让末影人无法拿起任何方块。[/size]

[size=16px]创建文件 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]minecraft/tags/blocks/enderman_holdable[/backcolor][/font][/size]

[code]{
    "replace": true,
    "values": [
    ]
}[/code]

[size=24px][b]§3 资源包[/b][/size]

[size=16px]资源包可以极大地提升原版模组的美观程度，因此现在大部分原版模组都需要使用配套的资源包。 资源包文件层次为[/size]

[code]resourcepacks
    资源包名称或资源包名称.zip
        pack.mcmeta
        pack.png
        assets
            命名空间
                lang
                    zh_cn.json
                    zh_tw.json
                    en_us.json
                models
                textures[/code]
[size=16px]其中 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]assets[/backcolor][/font] 的文件结构和 [url=thread-691700-1-1.html?cp=1]§1.4 文件结构[/url]中版本 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0].jar/assets[/backcolor][/font] 的结构是一样的。[/size]

[size=16px][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]assets[/backcolor][/font] 下所有文件和文件夹需使用小写英文、数字或-,_来命名，[b]不可使用大写字母[/b]，所有文本文件为使用 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]UTF-8[/backcolor][/font] 编码的 JSON文件，图片使用 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]png[/backcolor][/font] 格式。发布时可将所有内容压缩为一个 zip 文件。[/size]

[size=16px]压缩和解压的时候，注意文件层次，应当打开 zip 文件就可以看到数据包的 pack.mcmeta、pack.png 和 assets 文件夹。稳妥的做法是[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]打开文件夹->全选->右键->发送到->压缩文件夹[/backcolor][/font]。[/size]

[size=16px]在资源包菜单中，加载次序由下至上，因此上方的资源包内容会覆盖下方的资源包同名内容。由于原版模组往往会修改原版物品模型，因此同时安装多个原版模组的资源包时，很容易出现文件冲突，此时需要玩家手动合并资源包方可使用。具体而言，对比两个资源包的同名文件，将相同的模型文件的 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]overrides[/backcolor][/font] 合并，其它内容选择需要的留下即可。[/size]

[size=16px]在存档内的 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]resources.zip[/backcolor][/font] 资源包会在进入存档时自动加载(最后加载)。[/size]

[size=16px]本文中我们仅对资源包做简单介绍，更详细的文件说明请参阅[url=https://minecraft-zh.gamepedia.com/%E8%B5%84%E6%BA%90%E5%8C%85]资源包 - Minecraft Wiki，最详细的官方我的世界百科[/url]或论坛材质资源版相关内容。[/size]

[size=16px]材质包和资源包是不同的东西，目前 Minecraft Java 只有资源包，请不要使用错误的名称。[/size]

[size=20px][b]§3.1 pack.mcmeta 和 pack.png[/b][/size]

[size=16px]Minecraft 通过 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]pack.mcmeta[/backcolor][/font] 来识别资源包，因此该文件是不可或缺的。例如：[/size]

[code]{
    "pack": {
        "pack_format": 4,
        "description": "§6资源包名称\n§a简要介绍 §6by 某某作者"
    },
    "language": {
        "zh_cn":{
            "name": "简体中文",
            "region": "中国大陆",
            "bidirectional": false
        },
        "en_us":{
            "name": "English",
            "region": "United States",
            "bidirectional": false
        },
        "zh_tw":{
            "name": "繁体中文",
            "region": "中国台湾",
            "bidirectional": false
        }
    }
}[/code]
[size=16px][list]
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]pack_format[/backcolor][/font] 为资源包版本，必须为4。
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]description[/backcolor][/font] 为资源包描述，可以是单个的字符串或一个 raw json 对象。资源包菜单中可以看到。[/list][/size]
[size=16px]资源包可以包含一个 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]pack.png[/backcolor][/font]，它是大小为16*16的图片，用于在资源包菜单中显示。[/size]

[size=20px][b]§3.2 命名空间[/b][/size]

[size=16px]命名空间(namespace)为玩家自定义的、可操作的空间。不同资源包中相同的命名空间下相同的文件名内容会根据加载先后顺序被覆盖。特别地，原版内容被保存在 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]minecraft[/backcolor][/font] 命名空间，想要修改和替换原版的模型材质只需在你的数据包内建立 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]minecraft[/backcolor][/font] 命名空间和相应的同名文件并修改即可。[/size]

[size=16px]模型和材质的调用格式均为[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]命名空间:文件夹/文件名[/backcolor][/font]对应[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]命名空间/xxx/文件夹/文件名.后缀[/backcolor][/font]文件，[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]xxx[/backcolor][/font] 为 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]models[/backcolor][/font] 或 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]textures[/backcolor][/font]。[/size]

[size=16px]若命名空间为 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]minecraft[/backcolor][/font]，则可直接省略 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]minecraft:[/backcolor][/font]。[/size]

[size=16px]建议和数据包使用相同的命名空间名。[/size]

[size=20px][b]§3.3 语言文件[/b][/size]

[size=16px]语言文件可以放在任一命名空间下的 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]lang[/backcolor][/font] 文件夹下。语言文件中可以使用样式代码来实现彩色文字。[/size]

[size=16px]建议至少支持中文简体([font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]zh_cn[/backcolor][/font])、繁体([font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]zh_tw[/backcolor][/font])和英文([font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]en_us[/backcolor][/font])三种语言。建议所有物品、进度、游戏提示等文本均采用 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]translate[/backcolor][/font] 文本。[/size]

[size=16px]例如：[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]assets/命名空间/lang/zh_cn.json[/backcolor][/font][/size]

[code]{
    "item.cpp.magnet":"§r磁铁",
    ...
}[/code]
[size=16px][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]assets/命名空间/lang/en_us.json[/backcolor][/font][/size]

[code]{
    "item.cpp.magnet":"§rMagnet",
    ...
}[/code]
[size=16px]也可以用此方法修改物品的默认名称等内容。例如：[/size]

[size=16px][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]assets/命名空间/lang/zh_cn.json[/backcolor][/font][/size]

[code]{
    "item.minecraft.potion.effect.empty": "§r神秘药水",
    ...
}[/code]
[size=16px]则未知的药水的名称不再是[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]不可合成的药水[/backcolor][/font]，而是[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]神秘药水[/backcolor][/font]。[/size]

[size=16px][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]translate[/backcolor][/font] 还有个有趣但不一定实用的用法。[/size]

[code]{
    "The":"§a原版模组《更多的合成》已成功加载,",      
    "resourcepack":"版本",
    "does":"§a1.9.1§a.",
    "not":"§a更多内容请",
    "Install":"§e点击此处进入wiki.",
    "correctly":"§a作者:",
    "or":"§aruhuasiyu,",
    "Launch":"§aRubberTree",
    ...
}[/code]
[size=16px]则在数据包中输出相应文本时，若无资源包或资源包错误，则会显示[/size]

[code]The resourcepack does not Installl correctly or Launch.[/code]
[size=16px]以提示玩家资源包未加载。[/size]

[size=20px][b]§3.4 CustomModelData 物品模型[/b][/size]

[size=16px]由于1.14添加了 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]CustomModelData[/backcolor][/font]，我们可以拥有几乎无穷多的物品模型。为了便于他人整合，我们建议将 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]custom_model_data[/backcolor][/font] 的前四位确定模组的独有区段，后四位作为模组不同模型的值，例如 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]12970000-12979999[/backcolor][/font] 之间。此外，建议 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]custom_model_data[/backcolor][/font] 不要超过 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]16777216[/backcolor][/font]，原因见[url=https://www.mcbbs.net/thread-867051-1-1.html]custom_model_data的使用限制[/url]。[/size]

[size=16px]我们会给出一个例子来说明实现流程。[/size]

[size=16px]首先在 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]minecraft[/backcolor][/font] 下的胡萝卜钓竿物品模型中添加额外的模型([font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]overrides[/backcolor][/font])，并引用我们自定义的模型[/size]

[size=16px][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]assets/minecraft/models/item/carrot_on_a_stick.json[/backcolor][/font][/size]

[code]{
    "parent": "item/handheld_rod",
    "textures": {
        "layer0": "item/carrot_on_a_stick"
    },
    "overrides": [
        { "predicate": { "custom_model_data": 12970001 }, "model": "cpp:element/blue_force_of_sky"},
        { "predicate": { "custom_model_data": 12970002 }, "model": "cpp:element/green_force_of_water"},
        { "predicate": { "custom_model_data": 12970003 }, "model": "cpp:element/cyan_force_of_mountain"},
        { "predicate": { "custom_model_data": 12970004 }, "model": "cpp:element/orange_force_of_dirt"},
        { "predicate": { "custom_model_data": 12970005 }, "model": "cpp:element/yellow_force_of_earth"},
        { "predicate": { "custom_model_data": 12970006 }, "model": "cpp:element/red_force_of_fire"}
    ]
}[/code]
[size=16px]注意 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]custom_model_data[/backcolor][/font] 需要按照从小往大的次序，否则会导致后面的覆盖前面的。[/size]

[size=16px]然后创建自定义的模型[/size]

[size=16px][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]assets/cpp/models/element/red_force_of_fire.json[/backcolor][/font][/size]

[code]{
    "parent": "item/handheld",
    "textures": {
        "layer0": "cpp:element/red_force_of_fire"
    }
}[/code]
[size=16px]最后绘制材质，并将其保存为[/size]

[size=16px][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]assets/craftingpp/textures/element/red_force_of_fire.png[/backcolor][/font][/size]

[size=16px]这样我们就设计好了这个物品模型，在[url=thread-691700-1-1.html?cp=4]§4.4 物品设计[/url]我们将会说明如何使用该模型。[/size]

[size=20px][b]§3.5 模型[/b][/size]

[size=16px]模型本身只是一个 JSON 文件，只有 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]minecraft[/backcolor][/font] 命名空间下特定名称的文件才对应特定的物品/方块模型。例如 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]minecraft/models/item/carrot_on_a_stick.json[/backcolor][/font] 表示胡萝卜钓竿的物品模型，[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]minecraft/models/block/stone.json[/backcolor][/font] 表示石头的方块模型。[/size]

[size=16px]模型可以理解成一些可以旋转的长方体的拼接，每个长方体的若干个面贴上了材质图案。长方体的厚度可以为0，材质也可以只贴若干个面。[/size]

[size=16px]我们通过几个例子来理解下模型文件的语法。这个例子使用的是 Minecraft 自带的物品模型，并修改了在部分位置的显示：[/size]

[code]{
    "parent": "item/generated",
    "display": {

[size=16px][list]
[*][size=16px][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]parent[/backcolor][/font] 表示继承的模型，可以没有该项，若有则相当于将其文件内容复制到该位置。例如物品的平面模型、方块模型、玻璃板模型等，我们都可以直接调用而无需自己编写。或者需要多次使用的模型，我们也可以写好后在其它模型中直接调用。例子中继承的模型 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]minecraft:item/handheld[/backcolor][/font] 为手持物品的平面模型，这样我们可以免去调整物品在背包和展示框内的显示。[/size]
[*][size=16px][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]display[/backcolor][/font] 确定物品在不同位置[/size]

[size=16px][list]
[*][size=16px][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]ground[/backcolor][/font] 掉落物[/size]
[*][size=16px][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]head[/backcolor][/font] 头部[/size]
[*][size=16px][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]thirdperson_righthand[/backcolor][/font] 第三人称右手[/size]
[*][size=16px][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]thirdperson_lefthand[/backcolor][/font] 第三人称左手[/size]
[*][size=16px][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]firstperson_righthand[/backcolor][/font] 第一人称右手[/size]
[*][size=16px][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]firstperson_lefthand[/backcolor][/font] 第一人称左手[/size]
[*][size=16px][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]fixed[/backcolor][/font] 物品展示框[/size]
[*][size=16px][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]gui[/backcolor][/font] 背包或容器[/size]

[size=16px][list]
[*]放缩([font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]scale[/backcolor][/font])
[*]旋转([font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]rotation[/backcolor][/font])

[*][size=16px][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]elements[/backcolor][/font][/size]

[size=16px][list]
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]from[/backcolor][/font] 和 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]to[/backcolor][/font] 确定长方体的范围。[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]from[/backcolor][/font] 和 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]to[/backcolor][/font] 中的 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]x,y,z[/backcolor][/font] 范围为[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]-16[/backcolor][/font]到[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]32[/backcolor][/font]之间，不做放缩时，[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]0~16[/backcolor][/font]即一个方块的完整大小。
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]rotation[/backcolor][/font] 旋转这个长方体[size=16px][list]
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]origin[/backcolor][/font] 旋转中心
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]axis[/backcolor][/font] 沿哪个轴旋转
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]angle[/backcolor][/font] 旋转的角度，只能是[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]0, 22.5, 45, -22.5, -45[/backcolor][/font][/list][/size]
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]faces[/backcolor][/font] 为方块的6个面材质信息。我们放置方块时，离我们最近的为方块的北方(north)，然后站在方块的视角按前北后南(z轴)，左西右东(x轴)确定其6个面的朝向。[size=16px][list]
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]texture[/backcolor][/font] 确定了所使用的材质。我们建议按次序东北西南上下的次序写以便于想象。可以在材质文件中按照 [img]https://i.loli.net/2019/05/24/5ce786a746fe972611.png[/img] 的相对位置来绘制材质，也可以分为6个单独的材质。
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]uv[/backcolor][/font] 确定了选取材质的哪一部分。将材质划分为16×16个区域，该数值指定了便是所选的区域。例如材质文件为[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]64×64[/backcolor][/font]大小，则[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]"uv": [ 3.5, 3, 4, 4 ][/backcolor][/font]使用的是横向[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]14-16[/backcolor][/font]纵向[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]12-16[/backcolor][/font]的[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]2×4[/backcolor][/font]个像素。
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]rotation[/backcolor][/font] 旋转材质，值为[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]0,90,180,270[/backcolor][/font]。[/list][/size][/list][/size][/list][/size]
[size=16px]由于 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]scale[/backcolor][/font] 至多为[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]4[/backcolor][/font]，而 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]from to[/backcolor][/font] 至多为[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]-16[/backcolor][/font]到[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]32[/backcolor][/font]，因此模型最多可以放大至[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]12[/backcolor][/font]倍。想要更大的模型可以通过分段来旋转拼接而成，参阅[/size]

[size=20px][b]§3.6 材质[/b][/size]

[size=16px]材质的绘制需要用到诸如 Photoshop 之类的画图软件，具体请参考材质版或网络上的相关教程。[/size]

[size=16px]当模型调用的材质不存在时，会显示紫黑色的材质。如果模型是正确的，你仍然可以看出正确的模型样子。[/size]

[size=16px]材质可以是动态的。材质的高度为宽度的x倍时，动态材质可以有x帧，分别对应材质自上而下划分的x个块。动态材质还需要一个 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]mcmeta[/backcolor][/font] 文件，例如 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]foo.png[/backcolor][/font] 对应 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]foo.png.mcmeta[/backcolor][/font]，内容为[/size]

[code]{
      "animation": {
        "interpolate": false,
        "frametime": 4,
        "frames": [
            0,
            1,
            2,
            3,
            {"index":4,"time": 2},
            5
        ]
      }
}[/code]
[size=16px]这里[/size]

[size=16px][list]
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]interpolate[/backcolor][/font] 是否需要插值过渡。可以省略。
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]frametime[/backcolor][/font] 每一帧的默认时长(刻)。
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]frames[/backcolor][/font] 指定播放帧的次序[size=16px][list]
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]数字[/backcolor][/font] 从0开始
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]index[/backcolor][/font] 和 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]time[/backcolor][/font] 可以单独指定这一帧的时长(刻)[/list][/size][/list][/size]

[size=20px][b]§3.7 声音[/b][/size]

[size=16px]音效文件为 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]assets/命名空间/sounds.json[/backcolor][/font]，它调用的是 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]assets/命名空间/sounds[/backcolor][/font] 文件夹下的 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]ogg[/backcolor][/font] 文件。例如：[/size]

        "sounds": [
            "rf:lightsaber/swing1",
            "rf:lightsaber/swing2",
            "rf:lightsaber/swing3",

[size=16px]其中自定义名称下包括[/size]

[size=16px][list]
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]replace[/backcolor][/font] 是否覆盖原有的声音信息，默认为[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]false[/backcolor][/font]，即不覆盖。
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]sounds[/backcolor][/font] 音效文件列表，触发时会随机选择一个播放。

[size=20px][b]§3.8 字体[/b][/size]

[size=16px]添加 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]assets/minecraft/font/default.json[/backcolor][/font] 文件可以追加/覆盖特殊字符的字体显示，例如[/size]

[code]{
    "providers": [
        {
            "type": "bitmap",
            "file": "minecraft:font/mana0.png",
            "height": 91,
            "ascent": 85,
            "chars": ["\ue010"]
        },
        {
            "type": "bitmap",
            "file": "minecraft:font/mana1.png",
            "height": 91,
            "ascent": 85,
            "chars": ["\ue011"]
        }
    ]
}[/code]
[size=16px]我们追加了字符 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]\ue010[/backcolor][/font] 和 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]\ue011[/backcolor][/font] 的材质，其中[/size]

[size=16px][list]
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]file[/backcolor][/font] 指定了所用的材质，这里调用了 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]minecraft/textures/font/mana0.png[/backcolor][/font] 等材质
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]height[/backcolor][/font] 该字符的高度
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]ascent[/backcolor][/font] 该字符向上移动的长度
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]chars[/backcolor][/font] 对应的字符列表[/list][/size]
[size=16px]我们可以用 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]\u[/backcolor][/font] 开头来表示不容易打出的字符，参见[url=https://unicode-table.com]Unicode® Character Table[/url]。[/size]

[size=16px]利用这种方式，我们可以做到在成书、玩家快捷栏上方、聊天区域、物品名称等地方显示图片，见[url=https://www.mcbbs.net/thread-835539-1-1.html]【1.13】如何添加图片?[/url]。特别地，如果可以包含了使用负长度的空格的字体，还可以实现更为炫酷的效果，例如[url=https://www.bilibili.com/video/av67508247]字体黑科技 —— 潜影盒内容预览[/url]。[/size]

[size=24px][b]§4 模组规划与物品设计[/b][/size]

[size=16px]当我们有了一定的命令基础和数据包与资源包的相关知识之后，我们可以开始考虑做一个模组了。模组的目的是在原版的基础上做出一定的修改，模组的核心是内容，命令和技巧都只是为实现这一目的的手段。通常的设计路线如下图所示：[/size]

[size=16px][img]https://i.loli.net/2018/05/01/5ae7f0a5105f4.png[/img][/size]

[size=16px]平衡性可以从合成难度和触发条件等方面来调整。[/size]

[size=16px]原版模组不比基于 Forge/Liteloader API 的模组，在实现效率上一般会有所欠缺，因此我们在制作和测试过程中，应当优先保证流畅度，再考虑内容的充实性。[/size]

[size=16px]模组的整个生命周期中，所有内容都应当有完整的文档记录，以便于随时查看和修改。[/size]

[size=16px]为你的模组设定合理的版本管理方式，例如使用 Git 托管，有利于保存模组的所有历史版本。这可以减少因误操作导致的损失。[/size]

[size=20px][b]§4.1 名称设计[/b][/size]

[size=16px]模组的所有内容都应当被合理地命名，尽量采用简洁且有意义的命名，同时便于他人处理数据包冲突。这些内容包括数据包下所有的文件和文件夹名以及记分板、标签、组的名称等。建议各种名称均使用大驼峰、小驼峰或下划线记法。[/size]

[size=16px][list]
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]数据包名称[/backcolor][/font] 建议使用模组全名，空格和特殊字符使用下划线代替或忽略，也可使用中文名称。例如 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]craftingpp、更多的合成v1.10、CraftingPlusPlus[/backcolor][/font] 等。
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]命名空间[/backcolor][/font] 由于需要频繁使用，不建议使用过长的字符串，过短则容易冲突，可使用模组名称的全程或缩写来表示。命名空间必须由小写或下划线组成。建议整个模组使用一个命名空间，或若干相同前缀的命名空间。例如 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]craftingpp、cpp、fisma、mek[/backcolor][/font] 等。
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]物品名称[/backcolor][/font] 建议使用物品英文名称对应的小写+下划线写法作为物品id，具体设计见下一节。例如 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]cpp:red_force_of_fire[/backcolor][/font]。
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]记分板、组[/backcolor][/font] 建议使用命名空间为前缀的驼峰记法，因为记分板名称长度不可超过16。长度允许的话，使用下划线记法也可以。例如 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]cppValue、cppCraft、cppTicks、cpp_health[/backcolor][/font]。
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]实体标签[/backcolor][/font] 建议使用命名空间为前缀。例如 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]cpp_entities_checker、cpp_generate_dead、cpp_machines_marker、cppDarkAnimals[/backcolor][/font]。
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]tick函数[/backcolor][/font] 即 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]minecraft/tags/functions/tick.json[/backcolor][/font] 下添加的函数，我们建议使用 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]_main.mcfunction、tick.mcfunction、loop.mcfunction[/backcolor][/font] 等名称。有时候我们简称
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]load函数[/backcolor][/font] 即 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]minecraft/tags/functions/load.json[/backcolor][/font] 下添加的函数，我们建议使用 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]load.mcfunction、_init.mcfunction、root.mcfunction[/backcolor][/font] 等名称。
如果模组包含多个模块，可以在不同数据包下使用相同的命名空间、不同的文件夹，来分门别类。[/list][/size]

[size=20px][b]§4.2 调试[/b][/size]

[size=16px]加载中出现的资源包和数据包错误一般会在 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]logs/latest.log[/backcolor][/font] 中提示。测试时，打开这个文件，可以看到具体是哪个文件的哪个位置发生了错误。常见错误包括[/size]

[size=16px][list]
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]Unable to load model: '模型引用名' referenced from: 某物品#inventory: java.lang.IllegalStateException: Not a JSON Object: "parent"[/backcolor][/font] 该物品模型文件中调用的另一模型出错，错误出在"parent"
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]Using missing texture, unable to load[/backcolor][/font] 相应路径的材质不存在。
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]Couldn't load function at 函数路径 java.util.concurrent.CompletionException: java.lang.IllegalArgumentException: Whilst parsing command on line 行号 at position 列号[/backcolor][/font] 相应函数的相应位置出错。
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]Couldn't read function tag list 标签引用名 from 标签路径 in data pack 数据包[/backcolor][/font] 相应数据包的相应路径的标签出错，检查下是否调用了不存在的物品、方块或出错的函数，或是逗号使用错误。[/list][/size]
[size=16px]使用 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]/datapack list[/backcolor][/font] 来查看你的模组是否被识别了，如果压根没识别说明你的模组缺少 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]pack.mcmeta[/backcolor][/font] 或其错误。[/size]

[size=16px]如果进游戏异常卡顿，输入 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]/function 相应高频函数[/backcolor][/font]，如果提示执行了65535条函数，那么可能是函数使用了无限自我递归调用。检查相应的函数递归是否跳出条件有误。[/size]

[size=16px]调试时，可添加适当的 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]tellraw[/backcolor][/font] 命令来查看方块、实体、记分板值等，来确定何处出错。[/size]

[size=20px][b]§4.3 前置与附属[/b][/size]

[size=16px]使用他人已写好的前置可以在减少自己的工作量，例如[/size]

[size=16px][list]
[*][url=https://www.mcbbs.net/thread-900914-1-1.html]战利品表随机数[/url]
[*][url=https://www.mcbbs.net/thread-916294-1-1.html]方块通用处理[/url]
[*][url=https://github.com/ruhuasiyu/CraftingPlusPlus/tree/master/other_datapacks/%E5%90%88%E6%88%90%E9%80%9A%E7%94%A8%E5%A4%84%E7%90%86]合成通用处理[/url]
[*][url=https://github.com/ruhuasiyu/CraftingPlusPlus/tree/master/other_datapacks/%E5%8E%9F%E7%89%88%E6%A8%A1%E7%BB%84%E5%AE%B9%E5%99%A8%E6%89%A9%E5%B1%95]原版模组容器扩展[/url]
[*][url=https://www.mcbbs.net/thread-775667-1-1.html]【UIN】数据包——标签分类[/url]
等。这些前置已经包含了诸如生成随机数、处理模组方块、合成、容器等内容，这样开发者就不用再花费精力在这些事件的处理上，只需要调用它们提供的接口即可。[/list][/size]
[size=16px]建议在前置中添加函数[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]命名空间:datapack[/backcolor][/font][/size]

[code]scoreboard players set #datapack_命名空间 value 101[/code]
[size=16px]这样他人便可通过在[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]load函数[/backcolor][/font]中添加[/size]

[code]scoreboard players set #datapack_命名空间 value 0
function 命名空间:datapack
execute unless score #datapack_命名空间 value matches 101 run tellraw @a {"text":"[XX模组]：缺少必要的数据包前置[YY模组v1.0.1]！"}[/code]
[size=16px]以在缺少相应版本的前置时提醒使用者。[/size]

[size=16px]附属一般通过函数标签来实现。例如在你的模组中添加了一种机器，里面对物品进行了一些处理[/size]

[code]execute if block ~ ~ ~ barrel{Items:[{Slot:3b,id:"minecraft:mycelium"},{Slot:12b,id:"minecraft:dirt"}]} run replaceitem block ~ ~ ~ container.15 minecraft:mycelium
function #cpp:item_processer[/code]
[size=16px]那么其它开发者就可以通过在函数标签[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]#cpp:item_processer[/backcolor][/font]中添加相应的函数命令来实现更多的机器配方。[/size]

[size=20px][b]§4.4 物品设计[/b][/size]

[size=16px]由于原版模组从不添加原版不存在的物品，所以我们需要采取一些办法来区分。[/size]

[size=16px]例如：添加物品[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]红色火之力 (cpp:red_force_of_fire)[/backcolor][/font]，不可叠加，右键触发执行命令。我们可使用[/size]

[code]minecraft:carrot_on_a_stick{id:"cpp:red_force_of_fire",CustomModelData:12970013,display:{Name:'{"translate":"item.cpp.red_force_of_fire"}'}}[/code]
[size=16px]物品原型为胡萝卜钓竿，这样可以便于右键检测。[/size]

[size=16px][list]
[*]物品模组[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]id[/backcolor][/font]为[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]cpp:red_force_of_fire[/backcolor][/font]，使用模组对应的[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]命名空间:物品名称[/backcolor][/font]来表示，以与其它模组的物品区分。我们建议使用该[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]nbt[/backcolor][/font]来识别模组物品。
[*]使用胡萝卜钓竿的第[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]12970013[/backcolor][/font]个[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]custom_model_data[/backcolor][/font]模型。建议前4位数字为模组固定的编号，以与其它模组区分以便于整合，后4位表示该模组下同种原版物品对应的不同材质模型。在资源包中添加相应的模型。使用时注意[url=https://www.mcbbs.net/thread-867051-1-1.html]custom_model_data的使用限制[/url]。
[*]显示名称为 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]item.cpp.red_force_of_fire[/backcolor][/font] 对应的翻译文本。在资源包中添加相应的翻译文本以支持多种语言。
[*]添加模型
[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]assets/minecraft/models/item/carrot_on_a_stick.json[/backcolor][/font][code]  {
      "parent": "item/handheld_rod",
      "textures": {
          "layer0": "item/carrot_on_a_stick"
      },
      "overrides": [
          ...
          { "predicate": { "custom_model_data": 12970006 }, "model": "cpp:element/red_force_of_fire"},
          ...
      ]
  }[/code]
  [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]assets/cpp/models/element/red_force_of_fire.json[/backcolor][/font][code]  {
      "parent": "item/handheld_rod",
      "textures": {
          "layer0": "cpp:element/red_force_of_fire"
      }
  }[/code]
  并添加相应材质 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]assets/cpp/textures/element/red_force_of_fire.png[/backcolor][/font]。[/list][/size]
[size=16px]对于同类物品，我们可以添加额外的标签来注明。例如：使用[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]Byte 型[/backcolor][/font] nbt 来标记物品的类别，以便于探测；使用[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]List 型[/backcolor][/font] nbt 来标记诸如属性之类的内容，以及需要探测是否处于该类别的物品。[/size]

[size=16px]对于科技类的模组，我们推荐使用[url=https://www.mcmod.cn/post/927.html]原版模组矿物辞典[/url]标准来便于不同的原版模组/插件相互引用。[/size]

[size=16px]建议为模组添加的所有物品添加相应的战利品表来便于获取/修改/合成等。例如：[/size]

[size=16px][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]craftingpp/data/cpp/loot_tables/red_force_of_fire.json[/backcolor][/font][/size]

[code]{
    "pools": [
        {
            "rolls": 1,
            "entries": [
                {
                    "type": "item",
                    "name": "minecraft:carrot_on_a_stick",
                    "functions": [
                        {
                            "function": "set_nbt",
                            "tag": "{display:{Name:'{\"translate\":\"item.cpp.red_force_of_fire\"}'},id:'cpp:red_force_of_fire',CustomModelData:12970013}"
                        }
                    ]
                }
            ]
        }
    ]
}[/code]
[size=16px]烟火之星、皮革装备等受颜色控制而改变、具有双层模型的物品，若相应 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]CustomModelData[/backcolor][/font] 的模型也是双层的，是可以根据颜色而变化的。但是像盾牌受图案模式改变，其含有 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]CustomModelData[/backcolor][/font] 的对应物品并不会受到这些 nbt 影响。装备类物品穿戴后的显示效果也不受物品模型修改影响。 建议[/size]

[size=16px][list]
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]胡萝卜钓竿[/backcolor][/font]用于需要右键互动的的物品。
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]烟火之星[/backcolor][/font]用于无互动功能的物品。
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]树苗[/backcolor][/font]用于与土交互的(如花)。可以考虑移除或视觉上覆盖其原有模型。
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]小麦种子[/backcolor][/font]用于与耕地交互的(如作物)。可以考虑移除或视觉上覆盖其原有模型。
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]面包、曲奇[/backcolor][/font]用于食物模型。
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]蜜汁炖菜[/backcolor][/font]用于有药水效果的食物，注意会剩下碗。也可使用普通食物，配合进度来实现药水效果。
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]雕刻过的南瓜[/backcolor][/font]用于头饰，但是需要清除雕刻过的南瓜戴在头上的视野限制效果。你可能需要处理放置雕刻过的南瓜事件。
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]烟火之星、皮革盔甲、药水[/backcolor][/font]具有彩色效果，可用于不定色彩的物品。
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]兔子腿[/backcolor][/font]用于参与酿造的物品。
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]盔甲[/backcolor][/font]用于实现新盔甲，但是无法修改穿戴在身上时的显示材质。
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]工具和武器[/backcolor][/font]用于实现新工具和武器。
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]药水[/backcolor][/font]用于饮品。
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]物品展示框[/backcolor][/font]用于平面装饰。可以通过 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]EntityTag[/backcolor][/font] 标签来直接指定物品展示框的内含物，例如 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]item_frame{display:{Name:'{\"translate\":\"item.cpp.classical_painting\"}'},id:'cpp:classical_painting',CustomModelData:12970002,EntityTag:{Item:{id:'minecraft:firework_star',Count:1b,tag:{CustomModelData:12974001,isMachineBg:1b}},Tags:['cpp_item_frame_classical_painting','cpp_special_item_frame']}}[/backcolor][/font]
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]铁块和金块[/backcolor][/font]用于模拟矿物块，还可以参与信标基座。[/list][/size]
[size=16px]这些物品使用时均需考虑到物品本身的用途，利用或避免之。[/size]

[size=16px]自定义的玩家头颅的样式极为丰富，而且无需绘制材质即可使用，但是注意需要联网使用。以下两个网站列出了大量可用的头颅样式[/size]

[size=16px][list]
[*][url=https://heads.freshcoal.com/maincollection.php]Freshcoal[/url]
[*][url=https://minecraft-heads.com/custom/heads]Minecraft Heads[/url][/list][/size]
[size=16px]这些头颅类别丰富，且均为可放置的方块(玩家头颅)，因此在模组应用中十分常见。另外还可将图片上传至[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]minecraft.net[/backcolor][/font]域名下，以实现自定义头颅图案，参见[url=http://www.mcbbs.net/thread-824528-1-1.html]自定义方块(玩家头颅)[/url]。[/size]

[size=16px]自定义地图可用来显示自定义的图案，这个方法可避免使用资源包，但需将[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]data[/backcolor][/font]文件与数据包一同发布。首先确定好长宽比例，然后可使用[url=https://mc-map.djfun.de]MC Map Item Tool[/url]在线转换。注意这个网站生成的地图已经不符合1.14+版本的地图格式了，请在本地生成锁定图案的地图后将该网站生成的地图的[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]colors[/backcolor][/font]这个nbt复制过来。这些地图文件位于[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]世界名称/data[/backcolor][/font]文件夹。发布时，将其和数据包一同发布。使用如下命令获取相应地图：[/size]

[code]give @s minecraft:filled_map{map:10000}[/code]
[size=16px]选择负数的自定义地图的编号，或者手动将[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]data[/backcolor][/font]文件夹中的[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]idcounts[/backcolor][/font]调大，可避免和玩家在游戏内生成的地图编号冲突。[/size]

[size=20px][b]§4.5 发布[/b][/size]

[size=16px]当你设计并制作好全部内容且通过测试后，将你的数据包以及其它可能的内容，如地图、[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]data[/backcolor][/font]文件、资源包等一同发布。采用合适的版本号管理，并在发布时注明你所使用的命名空间、记分板、组、标签、地图区段、资源包等内容，以便于其他开发者整合或避免冲突。[/size]

[size=24px][b]§5 方块设计[/b][/size]

[size=16px]原版模组无法添加方块，一般的做法是修改方块物品的[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]CustomModelData[/backcolor][/font]的模型后，使用视线追踪法来判断放置的方块位置，并在方块位置生成盔甲架，并头戴相应物品。参考[/size]

[size=20px][b]§5.1 视线追踪法[/b][/size]

[size=16px]例如：放置[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]stone{id:"foo:bar"}[/backcolor][/font]。[/size]

[size=16px][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]foo/advancements/block/bar.json[/backcolor][/font][/size]

[code]{
    "criteria": {
        "put_furnace": {
            "trigger": "minecraft:placed_block",
            "conditions": {
                "item": {
                    "nbt":"{id:\"foo:bar\"}"
                }
            }
        }
    },
    "rewards": {
        "function": "foo:block/bar/reset"
    }
}[/code]
[size=16px][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]foo/functions/block/bar/reset.mcfunction[/backcolor][/font][/size]

[code]advancement revoke @s only foo:block/bar
execute anchored eyes run function foo:block/bar/ray
execute unless entity @e[type=area_effect_cloud,distance=..7,tag=foo_block_pos] positioned ~ ~1 ~ anchored eyes run function foo:block/bar/ray
execute unless entity @e[type=area_effect_cloud,distance=..7,tag=foo_block_pos] positioned ~ ~-1 ~ anchored eyes run function foo:block/bar/ray

[size=16px][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]foo/functions/block/bar/ray.mcfunction[/backcolor][/font][/size]

[code]execute if entity @s[distance=..7] positioned ^ ^ ^ if block ~ ~ ~ stone align xyz positioned ~0.5 ~ ~0.5 unless entity @e[type=minecraft:armor_stand,distance=..0.5,tag=foo_block] run summon minecraft:area_effect_cloud ~ ~ ~ {Tags:["foo_block_pos"]}

[size=16px][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]foo/functions/block/bar/put.mcfunction[/backcolor][/font][/size]

[code]summon armor_stand ~ ~ ~ {Invulnerable:1b,Invisible:1b,Small:1b,Marker:1b,NoGravity:1b,DisabledSlots:7967,Tags:["foo_block","foo_bar"]}

[size=16px]使用进度判断玩家放置了某物品，然后剥夺进度，递归获取该方块位置。如果视线上没有该方块，有可能是玩家将其对准下台阶或上台阶放置的，此时需要向上或向下平移一格检测。最后获取该方块位置，放置盔甲架，并将相应物品[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]loot[/backcolor][/font]进盔甲架头部位置。[/size]

[size=16px]注意到我们在视线追踪是进行了当前方块内是否已有[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]foo_block[/backcolor][/font]标签的盔甲架的判断。这在处理完整方块时是可以省略的，但是对于非完整方块，视线上可能已经有之前放置过的模组物品对应的盔甲架，使用该项判断可以避免在一个格子内放置两个盔甲架，而忽略掉后放的方块。[/size]

[size=16px]然而，这个做法仍然对其它模组的方块无效，因为你无法知道其它模组使用了何种标签。所以更严格的做法是判断当前方块处是否有头部佩戴拥有[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]tag.id[/backcolor][/font]物品的盔甲架。尽管这也不能保证万无一失，但确实可以降低冲突率。具体代码读者可自行实现。[/size]

[size=16px]最后，判断玩家主副手，并将相应的物品信息直接复制到盔甲架头部而非[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]loot[/backcolor][/font]固定的物品，可以在之后的破坏事件中保留原物品的额外信息。例如玩家对该物品进行了重命名。[/size]

[size=24px][b]§6 合成与烧炼配方[/b][/size]

[size=20px][b]§6.1 语法[/b][/size]

[size=16px]原版的合成、烧炼使用配方文件即可实现，较为简单，但是无法识别nbt，因此只能实现原版物品的合成和烧炼。同时模组若使用了相关物品，则相应的模组物品亦可代替原物品进行合成。[/size]

[size=16px]配方的通用语法为[/size]

[size=16px][list]
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]type[/backcolor][/font] 表示合成的类别[size=16px][list]
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]minecraft:crafting_shaped[/backcolor][/font] 工作台有序合成配方
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]minecraft:crafting_shapeless[/backcolor][/font] 工作台无序合成配方
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]minecraft:crafting_special_*[/backcolor][/font] 工作台特殊合成配方，处理皮革盔甲染色、烟火之星、地图复制等情形，不可手动配置。 
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]minecraft:stonecutting[/backcolor][/font] 切石机合成配方
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]minecraft:smelting[/backcolor][/font] 熔炉烧炼配方
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]minecraft:blasting[/backcolor][/font] 高炉烧炼配方
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]minecraft:smoking[/backcolor][/font] 烟熏炉烧炼配方
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]minecraft:campfire_cooking[/backcolor][/font] 营火烧炼配方[/list][/size]
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]group[/backcolor][/font] 字符串。具有相同该标识符的配方会在配方书中显示为一组。对特殊合成配方无效。
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]pattern[/backcolor][/font] 有序合成的合成样式，其中每个字符表示一个物品，空格表示空缺。可以比[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]3×3[/backcolor][/font]小。
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]key[/backcolor][/font] 有序合成的[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]pattern[/backcolor][/font]中字符与物品对应。可以只含一个[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]item[/backcolor][/font]或[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]tag[/backcolor][/font]；也可以是一个列表，列表中每个元素是一个[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]item[/backcolor][/font]或[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]tag[/backcolor][/font]。[size=16px][list]
[*]字符，可以代表一个物品或一个物品标签[size=16px][list]
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]item[/backcolor][/font] 对应的物品
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]tag[/backcolor][/font] 对应的物品标签[/list][/size][/list][/size]
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]ingredient[/backcolor][/font] 无序合成、切石机、烧炼的原材料。如果是切石机或烧炼，可以为这个列表中的任一物品或物品标签。如果是无序合成，配方就是这个列表中的每一个物品或物品标签。[size=16px][list]
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]item[/backcolor][/font] 对应的物品
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]tag[/backcolor][/font] 对应的物品标签[/list][/size]
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]result[/backcolor][/font] 合成的结果[size=16px][list]
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]item[/backcolor][/font] 物品名称
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]count[/backcolor][/font] 合成配方的物品数量，缺省为[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]1[/backcolor][/font]。切石机配方的物品数量，不可省略。[/list][/size]
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]experience[/backcolor][/font] 烧炼配方产生的经验值，可以为小数
[*][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]cookingtime[/backcolor][/font] 烧炼耗时(刻)。若缺省则使用默认的时间，熔炉烟熏炉为[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]200[/backcolor][/font]，高炉营火为[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]100[/backcolor][/font]。[/list][/size]

[size=20px][b]§6.2 有序合成[/b][/size]

[size=16px]例：使用8羊毛+鸡蛋有序合成两个羊刷怪蛋。[/size]

[size=16px][font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]cpp/recipes/sheep_spawn_egg.json[/backcolor][/font][/size]

[code]{
    "type": "minecraft:crafting_shaped",
    "pattern": [
        "###",
        "#*#",
        "###"
    ],
    "key": {
        "*": {
            "item": "minecraft:egg"
        },
        "#": {
            "tag": "cpp:wools"
        }
    },
    "result": {
        "item": "minecraft:sheep_spawn_egg",
        "count":2
    }
}[/code]
[size=16px]这里[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]cpp:wools[/backcolor][/font]为自定义的包含所有颜色羊毛的物品标签。[/size]

[size=20px][b]§6.3 无序合成[/b][/size]

[size=16px]例：2沙子+红石=2红沙。[/size]

[code]{

    "ingredients": [
        {
            "item": "minecraft:sand"

            "item": "minecraft:sand"

[size=20px][b]§6.6 覆盖原版配方[/b][/size]

[size=24px][b]§7.1 GUI 材质模型[/b][/size]

[size=16px]该模型用于机器的GUI，我们将其大小放大了12倍，也就是[font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]192×192[/backcolor][/font]大小。所以我们需要绘制这个大小的整数倍的材质。通过修改 [font=Monaco,Consolas,'Lucida Console','Courier New',serif][backcolor=#ebddc0]z[/backcolor][/font] 轴高度可调整材质的覆盖次序。[/size]

[size=24px][b]§7.9 管道[/b][/size]

[size=16px]科技类模组人们需要添加管道来传输电力、流体、物品等。物品管道是类似的，我们将物品的信息存储在管道上并传输即可，最终将其输入到容器中。常见的几种做法有：[/size]

[size=20px][b]§7.9.2 管道式 (WIP)[/b][/size]

[size=16px]优点是具有传统科技模组的风格，缺点是每个管道都是一个实体(仅有原版方块限制颇多)，管道多的时候会较为卡顿。[/size]

[size=16px]涌流式：能量从高处向周围比它低的管道/机器处流动，来最终实现能量传输。管道内会留有缓存。实现和无线传输类似，只是范围限定在1米内。[/size]

[size=16px]遍历式：从能量源出发，遇到过的管道做标记一直向下，如果遇到终点就回溯到分叉点，一直循环直到返回源头。[/size]

[size=20px][b]§9.2 投掷物[/b][/size]

[size=20px][b]§9.4 方块交互[/b][/size]

[size=16px]如果是与某些方块交互，可通过探测相应的交互判据的计分板即可。如果仅需要右击而不打开，可使用传送来实现强制关闭GUI。 [b]例[/b] 将附魔台、书架、发射器摆放成特定形状，在发射器中放入特定物品，在上方放置一个物品展示框并放入待添加效果的物品。手持魔杖右击发射器，然后等待一段时间将物品附加属性效果。星之魔杖给予更高的属性。[/size]

[size=16px]右键时，通过记分板探测到，然后探测玩家手持魔杖类型给予展示框不同标签。判断发射器物品和展示框物品是否符合要求，符合则进入计时，时间到了之后随机给予属性类别、数量、大小，清空发射器，删除标签。[/size]

[size=16px]将村民的{VillagerData:{profession:"minecraft:none"}可以确保村民不含交易。通过设置不同的VillagerData以及修改村民的头盔材质模型，可以将村民的外观和原版村民加以区分。例如([url=http://www.mcbbs.net/thread-772311-1-1.html]【活动】【原版模组】新春惊喜]。]http://www.mcbbs.net/thread-772311-1-1.html]【活动】【原版模组】新春惊喜]。[/url][/size]
